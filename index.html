Description

This is a SBT processor for generating files associated with the Lift framework. This was developed as part of my Google Summer of Code 2010 project. It's using a framework that I created, see: Lifty engine.

Using it

Start SBT in an existing project or create a new one and type the following in the regular SBT prompt:

*ScalaToolsSnapshot at http://scala-tools.org/repo-snapshots/
*lifty is org.lifty lifty 1.2

NB: The * (asterisk) is important, it tells SBT that it's a processor-related command. In the first line you're adding a repository and in the second you're declaring a processor. The first argument "lifty" is the name of the processor. You can pick any name you like. You will use this name to run the processor in the future.

Congratulations, you've now got a lift processor to make you more effective! You can now use the processor as you would any other SBT command.

Commands

The processor supports the following commands:

  • help: This will list all of the available commands
  • templates: This will list all of the available templates
  • create: This will attempt to create on of the templates

Examples:

lifty create snippet MySnippet or
lifty create snippet and it will request the necessary information
lifty create project-blank with user this will create both the project-blank and user templates (you can chain them!)

Templates

The processor can create the following templates:

  • Snippet: A snippet
  • Mapper: Model class using Mapper
  • Comet: Comet component
  • Layout: An empty layout with a surround tag
  • Project: A Lift project with some functionality to get you started
  • Project-blank: A blank Lift project that uses SBT as it's build system
  • User: Model class implemented using Mapper. It provides basic functionality that would expect from a User
  • Dependency Factory: I do not know.
